# Roadmap: De la Teoria a la Implementacion

## Filosofia del roadmap

No intentes construir todo a la vez. Cada fase produce un modelo funcional que puedes evaluar antes de anadir complejidad.

### Proyecto practico
Implementar un Transformer mini desde cero en PyTorch (solo decoder, ~10M params). Entrenarlo en un dataset pequeno (TinyStories, Shakespeare).

```python
# Lo minimo que debes ser capaz de implementar:
class MiniTransformer:
    def __init__(self):
        self.embedding = ...
        self.layers = [TransformerBlock() for _ in range(6)]
        self.head = ...
    
    def forward(self, tokens):
        x = self.embedding(tokens)
        for layer in self.layers:
            x = layer(x)  # attention + FFN
        return self.head(x)
```

### Proyecto practico
Anadir una TTT layer a tu modelo hibrido:

```python
class TTTLayer(nn.Module):
    """El hidden state es un mini-modelo que se actualiza por token"""
    def __init__(self, d_model, inner_dim=64):
        self.W = nn.Parameter(torch.randn(inner_dim, d_model) * 0.01)
        self.lr = nn.Parameter(torch.tensor(0.01))  # learning rate aprendible
    
    def forward(self, x):
        batch, seq_len, d = x.shape
        W = self.W.clone()
        outputs = []
        for t in range(seq_len):
            # Forward con pesos actuales
            out = W @ x[:, t]
            outputs.append(out)
            # Self-supervised update
            loss = self.self_supervised_loss(x[:, t], W)
            grad = torch.autograd.grad(loss, W)
            W = W - self.lr * grad
        return torch.stack(outputs, dim=1)
```

**Benchmark**: Evaluar si la TTT layer mejora en tareas que requieren adaptacion (few-shot, seguir instrucciones nuevas).

### Entregable
Modelo hibrido + TTT que demuestre capacidad de "aprender" durante inferencia.

### Proyecto practico
Anadir un modulo de memoria externa a tu modelo:

```python
class MemoryModule(nn.Module):
    def __init__(self, num_slots=1024, slot_dim=256, d_model=512):
        self.memory = nn.Parameter(torch.zeros(num_slots, slot_dim))
        self.query_proj = nn.Linear(d_model, slot_dim)
        self.write_proj = nn.Linear(d_model, slot_dim)
        self.erase_proj = nn.Linear(d_model, slot_dim)
        self.output_proj = nn.Linear(slot_dim + d_model, d_model)
    
    def forward(self, x, memory_state):
        # Read
        query = self.query_proj(x)
        weights = softmax(query @ memory_state.T / sqrt(d))
        read_out = weights @ memory_state
        
        # Write
        write_content = self.write_proj(x)
        erase = sigmoid(self.erase_proj(x))
        memory_state = memory_state * (1 - weights.T @ erase) + weights.T @ write_content
        
        # Combine
        output = self.output_proj(cat([x, read_out], dim=-1))
        return output, memory_state
```

**Benchmark**: Evaluar en tareas que requieren recordar informacion de muy atras en el contexto.

### Entregable
Modelo completo con backbone hibrido + TTT + MANN.

## Fase 5: Integracion y Evaluacion (2-3 semanas)

### Que hacer
1. Integrar todos los componentes
2. Entrenar end-to-end en un dataset significativo (SlimPajama, FineWeb)
3. Evaluar exhaustivamente

### Benchmarks criticos

| Benchmark | Que mide | Objetivo |
|---|---|---|
| Perplexity (val) | Calidad general | Comparable a Transformer del mismo tamano |
| MMLU (few-shot) | Conocimiento | Baseline razonable |
| Needle-in-haystack | Recall en contexto largo | Mejor que Mamba puro |
| Adaptation test | Aprende de ejemplos in-context | Mejor que Transformer (por TTT) |
| Memory test | Recuerda info de hace N tokens | Mejor que todos (por MANN) |
| Tokens/segundo | Velocidad de inferencia | Mejor que Transformer (estado fijo vs KV-cache) |

### Ablation studies
Quitar cada componente y medir el impacto:
- Sin capas de attention → recall baja?
- Sin TTT → adaptacion baja?
- Sin MANN → memoria a largo plazo baja?
- Sin SSM → velocidad baja?

## Consejo final

> Cada fase produce algo **funcional y evaluable**. No avances a la siguiente fase sin haber benchmarked la anterior. El progreso se mide en metricas, no en lineas de codigo.

Lo mas importante no es implementar todo — es entender **por que** cada componente aporta algo que los demas no pueden dar. Si entiendes eso, podras tomar decisiones de diseno informadas cuando surjan tradeoffs inevitables.
