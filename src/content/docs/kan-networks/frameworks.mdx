# Kolmogorov-Arnold Networks - Frameworks

## pykan (Implementacion oficial)

- **Repo**: https://github.com/KindXiaoming/pykan
- **Autor**: Ziming Liu (MIT, autor del paper)
- **Instalacion**: `pip install pykan`

```python
from kan import KAN

# Crear un KAN: [2 inputs, 5 hidden, 1 output]
model = KAN(width=[2, 5, 1], grid=5, k=3, seed=0)

# Entrenar
dataset = create_dataset(f=lambda x: torch.sin(x[:, 0]) + x[:, 1]**2, n_var=2)
model.train(dataset, opt="LBFGS", steps=20, lamb=0.01)

# Visualizar las funciones aprendidas
model.plot()

# Regresion simbolica: extraer formulas
model.auto_symbolic()
model.symbolic_formula()
# Output: sin(x_0) + x_1^2
```

### Funcionalidades clave de pykan
- **`model.plot()`**: Visualiza cada funcion aprendida en cada arista
- **`model.auto_symbolic()`**: Intenta convertir las splines a funciones simbolicas
- **`model.prune()`**: Elimina aristas innecesarias (sparsificacion)
- **`model.extend_grid(new_grid)`**: Refina el grid para mas precision

## efficient-kan (Optimizado para velocidad)

- **Repo**: https://github.com/Blealtan/efficient-kan
- **Ventaja**: 3-10x mas rapido que pykan
- **Instalacion**: `pip install efficient-kan`

```python
from efficient_kan import KANLinear
import torch.nn as nn

class EfficientKANModel(nn.Module):
    def __init__(self, in_features, hidden, out_features):
        super().__init__()
        self.kan1 = KANLinear(in_features, hidden, grid_size=5, spline_order=3)
        self.kan2 = KANLinear(hidden, out_features, grid_size=5, spline_order=3)
    
    def forward(self, x):
        x = self.kan1(x)
        x = self.kan2(x)
        return x

# Usa como cualquier modulo de PyTorch
model = EfficientKANModel(10, 64, 1)
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
```

### ChebyKAN
- **Idea**: Usar polinomios de Chebyshev como base
- **Ventaja**: Mejor aproximacion en intervalos acotados, propiedades de convergencia conocidas

### WavKAN (Wavelet KAN)
- **Idea**: Usar wavelets como funciones base
- **Ventaja**: Captura features a multiples escalas (multi-resolution)

### FastKAN
- **Repo**: https://github.com/ZiyaoLi/fast-kan
- **Idea**: Usa Radial Basis Functions en vez de B-splines
- **Ventaja**: Mas rapido y simple que B-splines

```python

```

## Ejemplo: KAN para Regresion Simbolica

```python
from kan import KAN
import torch

# Funcion objetivo: f(x,y) = exp(sin(pi*x) + y^2)
f = lambda x: torch.exp(torch.sin(torch.pi * x[:, 0]) + x[:, 1]**2)

# Dataset
dataset = {
    'train_input': torch.rand(1000, 2),
    'test_input': torch.rand(200, 2),
}
dataset['train_label'] = f(dataset['train_input'])
dataset['test_label'] = f(dataset['test_input'])

# Crear y entrenar KAN
model = KAN(width=[2, 3, 1], grid=5, k=3)
model.train(dataset, opt="LBFGS", steps=50, lamb=0.01)

# Visualizar funciones aprendidas
model.plot()

# Intentar regresion simbolica
model.auto_symbolic()
formula = model.symbolic_formula()
print(formula)  # Deberia aproximar exp(sin(pi*x) + y^2)
```

## Ejemplo: KAN como reemplazo de FFN en Transformer

```python
import torch
import torch.nn as nn
from efficient_kan import KANLinear

class TransformerBlockWithKAN(nn.Module):
    """Reemplaza el FFN del Transformer por KAN layers"""
    def __init__(self, d_model, n_heads, kan_hidden=256, grid_size=5):
        super().__init__()
        self.attention = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        
        # KAN en vez de FFN
        self.kan = nn.Sequential(
            KANLinear(d_model, kan_hidden, grid_size=grid_size),
            KANLinear(kan_hidden, d_model, grid_size=grid_size)
        )
    
    def forward(self, x):
        # Self-attention
        attn_out, _ = self.attention(x, x, x)
        x = self.norm1(x + attn_out)
        
        # KAN (en vez de FFN)
        kan_out = self.kan(x)
        x = self.norm2(x + kan_out)
        return x
```

## Tips Practicos

| Parametro | Recomendacion | Nota |
|---|---|---|
| Grid size | Empezar con 3-5, refinar a 10-20 | Mas grid = mas precision pero mas lento |
| Spline order (k) | 3 (cubico) | Casi nunca necesitas mas |
| Optimizer | LBFGS para pequenos, Adam para grandes | LBFGS converge mejor pero usa mas memoria |
| Regularizacion (lamb) | 0.001 - 0.01 | Regulariza la complejidad de las splines |
| Width | Mas estrecho que MLP equivalente | KANs necesitan menos neuronas |

### Cuando usar KAN vs MLP

| Usa KAN cuando... | Usa MLP cuando... |
|---|---|
| Necesitas interpretabilidad | Necesitas velocidad maxima |
| La funcion objetivo es suave | Los datos son ruidosos/discretos |
| Quieres regresion simbolica | Escalas a millones de parametros |
| Tienes pocas dimensiones | Tienes embeddings de alta dimension |
| Eficiencia de parametros importa | Tienes GPUs grandes |
